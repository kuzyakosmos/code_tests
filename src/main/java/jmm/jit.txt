javac компилирует .java --> .class файлы, содержащие байт-код (команды виртуальной машины JVM - универсальный промежуточный код для всех платформ)

Когда вы запускаете программу: java Main в дело вступает JVM (Java Virtual Machine)
читает .class файлы
интерпретирует байт-код
иногда компилирует части кода в машинные инструкции — входит JIT (

HotSpot profiler — это встроенный механизм в JVM (HotSpot), который собирает статистику о работе программы в реальном времени, чтобы JIT-компилятор мог принимать оптимизационные решения.
Он анализирует, какие части кода «горячие» (часто выполняются), и помогает JIT решать, что нужно компилировать и как оптимизировать.

JIT можно сказать кэширует участки в машинный код (оптимизированный - удаляет ненужные проверки, inline-ит методы и тд)

Java становится быстрее со временем, так как все больше участков кода попадают под JIT
